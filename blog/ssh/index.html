安全外壳协议 SSH

Homecat

SSH 为 Secure Shell 的缩写，由 IETF 的网络小组（Network Working Group）所制定；SSH 为建立在应用层基础上的安全协议。

SSH 命令

生成 SSH Key

ls -al ~/.ssh     //检查是否有存在的 SSH Key
ssh-keygen -t rsa -C <your_email>     //生成 SSH Key

说明：生成过程中，需要回答问题，一般都是”yes“通过；生成的 Key 在 ~/.ssh 目录(/home/用户名/.ssh)，目录中的 id_rsa 是私钥 id_rsa.pub 是公钥。

把私钥添加到本地高速缓存

ssh-add ~/.ssh/id_rsa

把公钥添加到网站

cat ~/.ssh/id_rsa.pub     //打开公钥并复制

以 Github 为例(Gitee基本相同)，在 github.com 注册用户成功后登录网站，找到用户头像打开菜单，依次点击 Settings > SSH and GPG Keys > New SSH Key；进入新增页面后，在 Title 栏中填入用户选定配对名称，在 Key 栏中粘帖公钥，点击 Add SSH Key，配对完成。

检验配对

ssh -T git@github.com

出现 “Hi ×××! You’ve successfully authenticated, but GitHub does not provide shell access.” 信息表示配对成功。
配对成功后，就可以上传和下载代码了。
